# Harness Lessons

이 문서는 하네스를 실제로 사용하며 발견한 반복 실수와 개선 규칙을 기록한다.

## 기록 형식

```markdown
## YYYY-MM-DD

- 문제: {어떤 응답이 부족했는가}
- 원인: {지시문 또는 스킬 기준의 빈틈}
- 수정: {앞으로 적용할 규칙}
```

## 2026-06-01

- 문제: Claude 기반 하네스는 `.claude` 구조에 묶여 있어 Codex Agent가 프로젝트 기준으로 바로 해석하기 어렵다.
- 원인: 루트 지시문과 스킬 위치가 Claude 관례에 맞춰져 있었다.
- 수정: `AGENTS.md`를 진입점으로 두고, Codex용 스킬 문서를 `skills/` 아래에 분리한다.

- 문제: Codex 전환 후 출력이 표와 기준 중심으로 바뀌어 Claude 버전의 쉬운 설명 느낌이 약해졌다.
- 원인: 스킬 출력 형식에 “먼저 쉽게 풀어 말하기” 단계가 없었다.
- 수정: 모든 학습 응답에서 쉬운 뜻, 상황 비유, 소리 느낌을 표보다 먼저 제공하도록 규칙을 추가한다.

- 문제: `もり`를 `も`, `り`로 나눠 각각 기억법을 붙이면 오히려 헷갈린다.
- 원인: 읽기 스킬이 글자별 표를 기본 출력으로 요구했다.
- 수정: 기본 암기법은 전체 단어 소리와 뜻을 하나의 장면으로 묶는다. 글자별 표는 사용자가 요청하거나 발음이 복잡할 때만 사용한다.

- 문제: `暑い(あつい)`의 끝 `い`처럼 문법 꼬리까지 암기법으로 만들면 핵심이 흐려진다.
- 원인: 모든 글자를 같은 중요도로 분해했다.
- 수정: 형용사 끝 `い`, 동사 어미 등은 핵심 암기법에서 제외하고 필요한 경우 문법 꼬리라고 짧게 설명한다.

- 문제: “쉬운 설명”만으로는 단어가 오래 기억되지 않을 수 있다.
- 원인: 암기법의 중심 원리가 명확히 경선식 영단어 스타일로 고정되어 있지 않았다.
- 수정: 발음과 뜻을 한국어 말장난/상황 이미지로 연결하는 경선식 암기문을 기본 출력으로 둔다.
