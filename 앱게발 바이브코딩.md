.# Vibe coding Web page

단순한 아이디어만으로 24분 만에 AI 서비스 완성하는 방법은? 아이디어를 사용자 관점에서 세분화하고, Claude Code와 InsForge 같은 AI 도구를 활용하여 기획부터 배포까지 전 과정을 효율적으로 진행하는 것이 핵심입니다.

## 1. 아이디어 하나로 24분 만에 AI 서비스 완성하기

단순한 아이디어만으로 24분 만에 AI 서비스를 완성하는 방법을 소개한다.

## 1.1. 아이디어 구체화 및 기획

!captureSource

1. **단순한 아이디어에서 시작한다**
    - 핵심 아이디어는 '바이럴 영상을 원클릭으로 만들어 주는 서비스'이다.
2. **사용자 관점에서 아이디어를 세분화한다**
    - 사용자가 서비스에 들어와서 할 일을 단계별로 분해한다.
        - 로그인
        - 프롬프트 입력
        - 영상 생성
        - 결과물 다운로드
    - 각 단계별로 필요한 기능을 도출한다.
        - 로그인 기능
        - 프롬프트 입력 페이지
        - 시작 이미지 필요성 인지 (이미지 생성 AI 필요)
3. **세분화된 사용자 흐름을 통해 필요한 기능들을 파악한다**
    - 사용자 흐름을 구체화하면 자연스럽게 필요한 기능들이 드러난다.
    - 잘게 쪼개진 아이디어는 실제 서비스 기획으로 발전한다.
4. **총 8개의 파트로 구성된 개발 과정을 살펴본다**
    - 파트 1: 사전 준비 작업 (Claude Code, Next.js 셋업)
    - 파트 2: 로그인 기능 구현
    - 파트 3: 이미지 생성
    - 파트 4: 영상 생성
    - 파트 5: 결제 기능 구현 (토스페이먼츠 사용)
    - 파트 6: 랜딩 페이지 디자인
    - 파트 7: 최종 배포
    - 파트 8: 마무리

## 1.2. Part 1: 개발 환경 세팅 및 사전 준비

!captureSource

1. **프로젝트 폴더 생성 및 기본 설정**
    - '모자이크'라는 이름으로 새로운 프로젝트 폴더를 생성한다.
    - **Claude Code**를 사용하여 개발을 진행하며, 유료 플랜을 사용한다.
    - 사용하는 도구보다 개발 흐름이 중요함을 강조한다.
2. **Next.js 프로젝트 생성 및 초기 파일 정리**
    - `npx create-next-app@latest` 명령어로 Next.js 프로젝트를 생성한다.
    - 기본 설정으로 프로젝트를 생성한다.
    - 개인 선호에 따라 불필요한 파일들을 정리한다.
        - `README` 파일 삭제
        - `next.config.ts` 파일에서 `images.unoptimized`를 `false`로 설정
        - `cloader.md` 파일 삭제 (Claude Code만 사용하므로)
    - `cloader.md` 파일에서 디자인을 위해 **Tailwind CSS**만 사용하도록 명시하고, 한국어 답변만 하도록 설정한다.
    - `public` 폴더 내부의 모든 파일을 삭제한다.
    - `app` 폴더의 `layout.tsx` 파일에서 폰트 관련 코드를 삭제하고, 타이틀과 설명을 '모자이크'로 수정한다.
    - `global.css` 파일에서도 폰트 관련 코드를 삭제한다.
    - `page.tsx` 파일의 모든 코드를 삭제하고 기본값으로 변경한다.
    - 이러한 파일 정리 작업은 필수는 아니며 생략 가능하다.
3. **GitHub 연동**
    - GitHub에서 '모자이크'라는 이름으로 새로운 저장소를 생성한다.
    - 생성된 저장소의 코드를 복사하여 Claude Code에 요청하여 GitHub 연동을 진행한다.
    - 연동 후 GitHub에서 새로고침하면 코드가 내부적으로 들어온 것을 확인할 수 있다.
    - GitHub 연동의 이점은 다음과 같다.
        - 변경 사항 확인 용이
        - 파트별 진행 시 체크포인트 역할
    - GitHub 연동은 거의 필수적이므로 적극 권장한다.
4. **Claude Code를 활용한 랜딩 페이지 초안 생성**
    - Claude Code 확장 프로그램을 설치하고 실행한다.
    - 서비스에 대한 간단한 설명과 색상 제약 조건을 Claude Code에 전달하여 랜딩 페이지 초안 생성을 요청한다.
    - 생성된 랜딩 페이지 초안을 개발 서버에서 확인한다.
    - 초안 디자인은 추후 수정될 예정이므로 현재 단계에서는 만족스럽지 않아도 괜찮다.
5. **변경 사항 커밋**
    - 개발 환경 설정, Next.js 셋업, GitHub 연결까지 완료되었음을 확인한다.
    - 소스 컨트롤 버튼을 눌러 변경 사항을 확인한다.
    - 메시지에 '파트원 완성'이라고 작성하고 커밋한다.
    - 싱크 버튼을 눌러 GitHub에도 변경 사항을 반영한다.
    - GitHub에서 변경 이력을 확인할 수 있다.

## 1.3. Part 2: 로그인 기능 구현 (InsForge 활용)

!captureSource

1. **InsForge 소개 및 특징**
    - 기존에 사용하던 Superbase 대신 **InsForge**라는 새로운 백엔드 서비스를 사용한다.
    - InsForge는 AI 에이전트를 위해 설계된 백엔드 서비스로, 데이터베이스, 인증, 스토리지, AI 연동 및 배포 기능을 제공한다.
    - InsForge의 가장 큰 차이점은 **AI 에이전트가 직접 모든 작업을 수행**할 수 있도록 설계되었다는 점이다.
        - 개발자가 직접 대시보드에서 수정할 필요 없이 AI 에이전트에게 지시하면 알아서 처리한다.
    - InsForge는 오픈 소스이며 GitHub에서 소스 코드를 확인할 수 있고, Docker를 사용한 셀프 호스팅도 가능하다.
    - 무료 요금제로도 충분히 사용할 수 있다.
2. **InsForge 프로젝트 생성 및 Claude Code 연동**
    - InsForge에 접속하여 회원 가입 후 '모자이크'라는 이름으로 프로젝트를 생성한다.
    - 가장 가까운 리전인 싱가포르를 선택한다.
    - 프로젝트 생성 후 대시보드에서 'Claude Code 옵션'을 선택하여 설정 프롬프트를 복사한다.
    - 복사한 프롬프트를 Claude Code에 붙여넣어 InsForge와 Claude Code를 연동한다.
    - 연동이 완료되면 대시보드에 자동으로 반영된다.
    - 이제 Claude Code에게 테이블 생성, 인증 설정 등을 지시하면 InsForge를 통해 직접 실행된다.
3. **Google 로그인 기능 구현**
    - InsForge를 사용하여 Google 로그인을 구현하도록 Claude Code에 요청한다.
    - 로그인 페이지와 회원가입 페이지를 분리하지 않고 'Auth 페이지' 하나만 생성하도록 요청한다.
    - 로그인한 사용자의 정보를 저장할 **User 테이블**도 동시에 설정하도록 요청한다.
    - 구현 완료 후 웹사이트에서 Google 로그인 페이지가 완성된 것을 확인한다.
    - Google 로그인을 진행하면 사용자 정보가 InsForge 대시보드의 User 테이블에 잘 기록되는 것을 확인한다.
4. **로그인 페이지 디자인 수정**
    - 상단의 불필요한 헤더 부분을 삭제한다.
    - 피그마로 감싸진 부분을 삭제하고, 실제 서비스 로고로 변경한다.
    - 미리 준비된 로고 파일을 `public` 폴더에 넣고 Claude Code에 요청하여 적용한다.
5. **Google 로그인 API 설정 (배포 환경 대비)**
    - InsForge 인증 탭에서 Google 로그인 설정을 편집한다.
    - 현재는 테스트 환경이므로 공개 키(Public Key)를 사용하지만, 배포 환경에서는 사용할 수 없으므로 비공개 키(Client ID, Client Secret)를 발급받아야 한다.
    - Google Cloud Console에서 '모자이크' 프로젝트를 생성하고 사용자 인증 정보 만들기를 진행한다.
    - 동의 화면 구성을 완료하고, 유형을 'Web'으로 선택하여 클라이언트 ID와 클라이언트 시크릿을 발급받는다.
    - 승인된 JavaScript 원본에는 테스트 환경인 `http://localhost:3000`을 입력한다.
    - 승인된 리디렉션 URI에는 InsForge에서 제공하는 값을 복사하여 붙여넣는다.
    - 발급받은 클라이언트 ID와 클라이언트 시크릿을 InsForge에 등록한다.
    - 이로써 실제 Google 로그인 연동이 완료된다.
6. **변경 사항 커밋**
    - 로그인 기능 구현 및 디자인 수정, Google 로그인 API 설정까지 완료한 후 변경 사항을 커밋한다.

## 1.4. Part 3: 이미지 생성 기능 구현

!captureSource

1. **핵심 플로우 설계**
    - 사용자는 영상 생성을 요청하지만, 내부적으로는 **이미지 생성**과 **영상 생성** 두 단계로 나뉜다.
    - 영상 생성 AI는 시작 이미지를 입력해야 퀄리티가 높아지므로, 이미지를 먼저 생성하는 흐름을 따른다.
    - 핵심 플로우: 프롬프트 입력 → 이미지 생성 → 영상 생성
2. **Generate 페이지 UI 구현**
    - Claude Code에게 영상 생성을 위한 새로운 페이지인 **Generate 페이지**를 만들도록 요청한다.
    - 이 페이지는 로그인한 사용자만 접근 가능하도록 설정한다.
    - 페이지 UI에는 네 가지 영상이 필요하므로, `public/videos` 폴더에 예시 영상 4개를 배치한다.
    - 생성된 Generate 페이지로 이동하여 기본적인 UI가 잘 구현되었는지 확인한다.
    - UI에서 영상이 가려 보이는 부분을 수정 요청한다.
3. **Replicate를 활용한 이미지 생성 로직 구현**
    - 이미지 생성 및 영상 생성을 위해 **Replicate** 서비스를 사용한다.
        - Replicate는 다양한 AI 모델을 API로 호출할 수 있게 해주는 서비스이다.
    - 이미지 생성에는 **GPT Image 2 모델**을 사용한다.
    - Replicate에서 API 토큰을 발급받아 `.env` 파일에 환경 변수로 저장한다.
        - 환경 변수는 절대 노출되지 않도록 주의한다.
    - Claude Code에게 이미지 생성 로직 구현을 요청한다.
        - 사용자가 사진을 첨부하면 이미지가 생성되는 기능을 구현한다.
    - 생성된 이미지를 저장하기 위해 InsForge에 새로운 **Storage**를 만들고, 관련 기록을 저장할 **Generation 테이블**도 생성하도록 요청한다.
    - Replicate에서 GPT 모델의 API 정보를 복사하여 프롬프트에 포함시킨다.
    - 이미지 생성, InsForge 스토리지 업로드, Generation 테이블 기록 저장까지 한 번에 지시한다.
    - 테스트 결과, 이미지가 잘 생성되고 InsForge 스토리지 및 데이터베이스에 기록이 저장되는 것을 확인한다.
4. **변경 사항 커밋**
    - 이미지 생성 기능 구현 및 관련 데이터베이스, 스토리지 연동까지 완료한 후 변경 사항을 커밋한다.

## 1.5. Part 4: 영상 생성 기능 구현

!captureSource

1. **Replicate를 활용한 영상 생성 로직 구현**
    - 영상 생성에도 Replicate를 사용하며, **CDN 2.0 모델**을 활용한다.
        - CDN 2.0은 현재 성능이 가장 좋다고 알려져 있다.
    - 핵심은 이전에 생성한 이미지를 **시작 프레임**으로 넣어 이미지 스타일과 구도를 유지하면서 움직임을 시작하는 영상을 만드는 것이다.
    - Claude Code에게 영상 생성 기능 구현을 요청한다.
        - 생성된 영상을 InsForge 스토리지에 저장하고 Generation 테이블에 기록하도록 요청한다.
    - Replicate에서 CDN 2.0 모델의 API 정보를 복사하여 프롬프트에 포함시킨다.
    - 테스트 결과, 사진을 바탕으로 이미지를 만들고 그 이미지에서 영상까지 생성하는 전체 파이프라인이 구현되었음을 확인한다.
    - 데이터베이스와 스토리지에도 결과물이 잘 저장되는 것을 확인한다.
2. **생성 결과물 확인 및 다운로드 기능 구현**
    - 사용자가 생성한 영상 목록을 보여주고, 각각을 다시 보거나 다운로드할 수 있도록 구현한다.
    - 썸네일이 작게 보이고 마우스를 올리면 확장되어 자세히 확인할 수 있으며, 클릭 시 다운로드 가능한 UI를 요청한다.
    - 좌측 로고를 실제 로고로 교체하고 히스토리 기능을 깔끔하게 구현한다.
3. **바이럴 영상 프롬프트 추가**
    - 네 가지 케이스 (야구, 아이돌, 쇼미, 스트릿)에 대한 바이럴 영상 프롬프트를 추가한다.
    - 각 프롬프트에 맞춰 자신의 사진을 넣어 테스트를 진행한다.
4. **영상 생성 모델 변경 (CDN 2.0 → Stable Diffusion XL Video)**
    - CDN 2.0 모델에서 사람을 넣었을 때 위험하다고 판단하는 경우가 많아, 영상 생성 모델을 **Stable Diffusion XL Video (SDXL Video)**로 변경한다.
    - 모델 교체 과정은 동일하게 Replicate에서 필요한 정보를 복사하여 적용한다.
    - 새롭게 생성된 네 가지 영상으로 교체하고 새로고침하여 결과를 확인한다.
    - 결과물이 다소 아쉽지만, 프롬프트를 다듬으면 SNS에 올릴 수 있을 수준이라고 평가한다.
5. **변경 사항 커밋**
    - 영상 생성 기능 구현까지 완료한 후 변경 사항을 커밋한다.

## 1.6. Part 5: 결제 기능 구현 (토스페이먼츠)

!captureSource

1. **결제 모델 설계**
    - 서비스의 결제 모델은 **크레딧 기반 1회성 결제**로 구현한다.
    - 구독 요금제보다는 사용할 만큼 결제하는 크레딧 시스템이 바이럴 영상 제작 서비스에 적합하다고 판단한다.
    - 가격 구조 예시: 10 크레딧에 4,900원, 25 크레딧에 9,900원.
        - 실제 런칭 시에는 API 비용을 계산하여 가격을 조정해야 한다.
2. **토스페이먼츠 연동**
    - 결제 시스템으로 **토스페이먼츠**를 사용한다.
        - 한국에서 결제를 연동하기에 가장 편리한 서비스이다.
    - 문서가 잘 되어 있어 간편하게 구현할 수 있다.
    - 실제 결제를 받으려면 사업자 등록과 카드사 심사가 필요하며, 약 한 달이 소요된다.
    - 토스페이먼츠 연동 시 연회비도 고려해야 한다.
    - 구현 방법 자체는 테스트 모드와 실제 모드가 동일하다.
3. **토스페이먼츠 MCP 서버 및 LLM 퀵 레퍼런스 연동**
    - 토스페이먼츠 개발자 센터에서 'AI로 시작하기'를 누르고 MCP 서버를 연동한다.
    - LLM 퀵 레퍼런스 페이지 전체를 복사하여 Claude Code에 전달한다.
    - 사전 작업을 완료한 후 결제 기능 구현을 요청한다.
4. **결제 기능 구현 및 데이터 저장**
    - 사용자가 영상을 생성할 때 1 크레딧이 소모되도록 설정한다.
    - 토큰 결제를 위해 토스페이먼츠 1회성 결제를 연동한다.
    - 사용자가 구매한 토큰과 결제 내역을 InsForge 데이터베이스에 저장하도록 한다.
    - 토스페이먼츠 개발자 센터에서 API 키를 발급받아 `.env` 파일에 저장한다.
    - 결제 위젯이 깔끔하게 생성되고, 결제 완료 시 크레딧이 추가되는 것을 확인한다.
    - 데이터베이스와 User 테이블의 크레딧 정보도 정상적으로 업데이트되는 것을 확인한다.
5. **변경 사항 커밋**
    - 토스페이먼츠 연동 및 결제 기능 구현까지 완료한 후 변경 사항을 커밋한다.
